# master_harness

A Wails desktop app (Go backend, React frontend) built with hexagonal
architecture: every technology sits behind an interface, and only `wire.go`
knows which implementation fills which interface.

## Layout

```
main.go                        entrypoint: reads config, starts the app builder
wire.go                        composition root: builds implementations, connects interfaces
config.yaml                    app configuration (name, window size, background, log level)
Makefile                       dev / build / run / test commands
internal/
  interface/                   ports — pure contracts, no tech
    input/                     flowing in:  Config
    output/                    flowing out: Logger, AppBuilder
  implementation/              one package per technology
    input/
      config/viper/            Config from config.yaml (viper)
      config/env/              Config from environment variables
      wailsapi/                API bound to the frontend — JS calls these methods
    output/
      logger/slog/             Logger on stdlib log/slog
      app_builder/wails/       AppBuilder on Wails
    helpers/                   small shared utilities
frontend/                      React 19 + TypeScript + Vite + Tailwind v4 + shadcn/ui
  src/App.tsx                  UI calling Go via generated bindings
  wailsjs/                     bindings generated by wails build
```

The rules:

- `interface/` knows nothing about any technology.
- `implementation/` packages depend on `interface/`, never on each other.
- `wire.go` is the only place a concrete implementation is chosen.

## Commands

```sh
make dev     # start with hot reload (recommended for development)
make run     # build the production .app and launch it
make build   # build the production .app only
make test    # run Go tests
```

Note: the app reads `config.yaml` from the working directory, so run it from
the repo root.
